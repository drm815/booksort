# 책정리 도우미 — 설계 문서

**작성일:** 2026-05-12  
**프로젝트:** booksort  
**대상:** 학교 도서관 도서부 학생 및 사서 선생님

---

## 1. 개요

학교 도서관 장서 정리를 돕는 모바일 웹앱. 책에 붙은 대출 바코드를 스캔하면 KDC(한국십진분류법) 청구기호 기준으로 해당 책의 앞뒤 3권씩을 선반 시각화로 표시해준다. 도서목록은 구글 시트에서 실시간으로 불러온다.

---

## 2. 사용자

- **도서부 학생:** 스마트폰으로 바코드 스캔 → 어디에 꽂을지 확인
- **사서 선생님:** 구글 시트에서 도서목록 관리, 앱은 항상 최신 데이터 반영

---

## 3. 핵심 기능

### 3.1 바코드 스캔
- 카메라로 대출 바코드 직접 스캔 (`@zxing/library`)
- 등록번호 직접 입력 fallback
- 탭으로 카메라/입력 전환
- 최근 스캔 기록 표시 (세션 내)

### 3.2 앞뒤 책 표시 (핵심)
- 스캔된 등록번호로 도서목록에서 해당 책 조회
- 청구기호(KDC) 기준으로 전체 목록 정렬
- 현재 책의 앞 3권 + 현재 책 + 뒤 3권 = 총 7권 표시
- 각 책의 **자료상태** (대출가능 / 대출중) 표시
- 선반 시각화: 책꽂이처럼 가로로 나열, 현재 책 위치에 "여기!" 슬롯 강조
- 하단에 바로 앞/뒤 책 이름 텍스트로도 명시

### 3.3 도서목록 연동
- 구글 시트 공개 CSV URL을 환경변수(`VITE_SHEET_CSV_URL`)로 고정
- 앱 시작 시 자동 fetch → 브라우저 메모리 + `localStorage` 캐시
- 수동 새로고침 버튼 (우상단)
- 캐시 유효 시간: 30분 (30분 경과 시 자동 재fetch)
- 최근 스캔 기록: 세션 내 메모리에만 저장, 페이지 새로고침 시 초기화

## 6. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Vite + React | 빠른 빌드, 컴포넌트 기반 |
| 바코드 스캔 | `@zxing/library` | 웹 카메라 바코드 인식 표준 라이브러리 |
| 스타일 | Tailwind CSS | 모바일 반응형 빠른 구현 |
| 데이터 | Google Sheets CSV (공개) | 별도 서버 불필요 |
| 상태 관리 | React 기본 (useState/useContext) | 단순한 구조에 충분 |
| 캐시 | localStorage | 오프라인 fallback + 재방문 속도 개선 |
| 배포 | Vercel | 무료, GitHub 자동 배포 |

## 7. 데이터 흐름

```
앱 시작
  └→ localStorage 캐시 확인
       ├→ 캐시 유효 (30분 이내): 캐시 데이터 사용
       └→ 캐시 없음 / 만료: CSV URL fetch
            └→ CSV 파싱 → 메모리 + localStorage 저장

바코드 스캔
  └→ 등록번호 추출
       └→ 메모리 내 도서목록에서 등록번호 검색
            ├→ 찾음: 청구기호로 정렬된 위치 계산 → 앞뒤 3권 추출 → 결과 화면
            └→ 못찾음: "등록번호를 찾을 수 없습니다" 안내
```

---

## 8. KDC 정렬 방식

청구기호는 문자열 순 정렬이 아닌 KDC 규칙을 따른다:

1. 분류번호 (숫자 부분): 수치 기준 정렬 (예: `813.6` < `813.61`)
2. 저자기호 (문자+숫자 부분): 문자열 기준 정렬
3. 동일 청구기호 내 등록번호 순 정렬 (같은 책 여러 권 처리)

## 10. 비기능 요구사항

- **모바일 최적화:** 세로 방향 기준, 터치 친화적 UI, 최소 탭 크기 44px
- **로딩 상태:** CSV fetch 중 스피너 표시
- **에러 처리:** 네트워크 오류 시 캐시 데이터로 fallback + 경고 표시
- **접근성:** 한국어 UI, 명확한 색상 대비 (대출가능: 초록, 대출중: 빨강)
- **성능:** CSV 파싱은 Web Worker로 처리 (3,000~5,000권 기준)
